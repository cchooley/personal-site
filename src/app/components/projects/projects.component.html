VYP - Seussical, The Musical!

Built using: Angular, TypeScript, PostgreSQL, Knex.js, Express.js, Heroku, Firebase, CSS, HTML

This is a full-stack app that was built for Aurora, CO's Vintage Theatre, one of the largest non-profit live theatres in Colorado.

Vintage required a means of tracking the registry for their Youth Program's summer camp, which accepts both paid enrollees as well as scholarship students.

Using PostgreSQL, Knex.js, and Express.js, I built an API that would offer persisting registration data on enrollees and record whether they are paid students or scholarship recipients.

On the frontend, I built a single page application with Angular that offers all relevant program information, as well as relevant forms for paid vs. scholarship. It also has an "Admin" section that allows administrators to track enrollment in all forms.

Check out VYP!

Still needed: I hope to soon integrate Stripe to capture payments as well as a feature like OAuth that will allow for administrators to log in and access the "Admin" section (and, of course, to keep the general public from accessing it!)

GOGOT - The Game of Game of Thrones

Built using: Angular, TypeScript, Firebase, CSS, HTML

This app is demos a crucial aspect of a larger, long-term project called GOGOT that I am working on another colleague. We came up with the idea for this game before Game of Thrones’ fifth season that kind of caught fire - essentially formatting the “Survivor Pool” NFL game to apply to HBO's Game of Thrones.

The GOGOT app, including this demo version, is chiefly designed to administer and record weekly selections. This would be something that could identify a player, present them with the eligible characters for selection, log their pick and display the week’s selections. All in an efficient and stylish way.

My objective for this project was to use Angular to code the aforementioned game logic into a workable demo, and in those terms it functions exactly how I’d like.

Check out GOGOT!

Still needed: A final, truly-playable version will require components that will create and store player accounts, store weekly pick information for characters and players alike, and conditionally open and close the weekly selections at the correct times (last season the pick period typically ran from Tuesday or Wednesday afternoon through the Sunday evening start time of that week’s episode). I’d also love to add a message board for trash talk and some other peripheral aspects that players can geek out on - episode info, memes, quotes, etc.

Cannalog

Built using: Angular, TypeScript, PostgreSQL, Knex.js, Express.js, Heroku, Firebase, CSS, HTML

This full-stack app is something I came up with alongside 3 other Galvanize students during a school-wide hackathon.

Given our 5-hour time constraints, we opted on broad, crowd-pleasing (ahem) content and created a site that would allow users to browse a selection of fine cannabis. The rubric required the site to have CRUD functionality that would allow users to get all of the weed, add an entry of their own, edit the entries, and delete an entry as well.

We went a step further and added features that would filter out the various strains by type, as well as generate a random strain at the click of a button.

My teammates created the backend, chiefly by designing the database that contains the information being pulled by the front-end.

I built the front-end with Angular, with the main focus being giving full CRUD functionality.

Aside from a bit of superficial styling changes I recently made, this is essentially what we were able to come up with after 6 hours of hard coding. Not a bad way to spend the day!

Check out Cannalog!

Still needed: So many things. But in the spirit of showing what I'm capable of in a short timeframe, as well as fleshing this section of the site via adhering to the rule of threes, I'm leaving this as is. For now.
